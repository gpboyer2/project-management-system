# CSSC Node-View 项目功能点汇总（API接口需求）

> 更新时间：2025年11月24日 12:10
> 项目类型：基于Node-RED的可视化流程设计与系统测试平台
> 技术栈：Vue 3 + TypeScript + Vite + LogicFlow + Element Plus
> 目的：为后端API接口开发提供完整的功能需求支撑

**说明**：本文档仅列出需要后端API接口支持的功能点，纯前端交互功能（如UI切换、拖拽、展开收起等）不在此列

## 1. 系统架构功能

### 1.1 基础框架
- 现代化前端架构：基于Vue 3 Composition API + TypeScript
- 快速构建系统：使用Vite 4.5.0实现极速热重载
- 组件化UI：Element Plus 2.0.4 提供完整的界面组件
- 状态管理：Pinia 2.1.0 + 持久化插件实现全局状态管理
- 路由系统：Vue Router 4.2.0 提供单页应用导航
- 响应式设计：支持多种屏幕尺寸的自适应布局

### 1.2 主题系统
- 双主题支持：亮色主题 / 暗色主题一键切换
- CSS变量系统：统一的色彩、间距、字体、阴影设计规范
- 主题持久化：用户主题选择自动保存到本地存储
- 动态样式更新：主题切换时实时更新所有组件样式

### 1.3 布局系统
- 基础布局组件：顶部导航栏 + 主要内容区域
- 用户中心：头像下拉菜单，支持用户管理、体系配置、退出登录
- 导航菜单：清晰的页面层级导航结构
- 面包屑导航：多层级页面快速定位和返回

## 2. 核心功能模块

### 2.1 流程图设计 (flowchart)

#### 2.1.1 节点调色板功能
- 节点分类管理：多种节点类型的分类展示
- 智能搜索：实时搜索节点类型，快速定位目标节点
- 分类折叠：支持展开/收起节点分类，优化界面空间
- 拖拽创建：从调色板拖拽节点到画布创建新节点

#### 2.1.2 画布编辑功能
- 节点选择：单击选择节点，支持多选操作
- 节点拖拽：在画布内自由拖拽节点位置
- 连接管理：节点间创建、编辑、删除连接关系
- 网格对齐：可选的网格显示，辅助节点对齐
- 缩放控制：画布缩放功能，适应不同复杂度的流程图

#### 2.1.3 工具栏操作
- 保存流程图：保存当前流程图配置到服务器
- 导出功能：将流程图导出为JSON格式文件
- 导入功能：从文件或文本导入流程图配置
- 全选操作：选择画布中的所有节点
- 自动布局：一键自动排列节点位置
- 清空画布：清除所有节点和连接
- 网格切换：显示/隐藏网格对齐线

#### 2.1.4 属性编辑面板
- 面板显示控制：右侧属性面板的显示/隐藏切换
- 面板宽度调整：拖拽调整属性面板宽度
- 节点属性编辑：修改选中节点的配置参数
- 实时更新：属性修改实时反映到节点显示

### 2.2 系统级设计 (system-level-design)

#### 2.2.1 树形结构管理
- 三级层次模型：系统级 → 设备级 → 软件级的三层架构
- 搜索功能：按项目、设备、软件名称进行搜索
- 节点展开/收起：点击展开或收起树节点
- 根节点创建：新增项目级别的根节点

#### 2.2.2 右键上下文菜单
- 节点编辑：进入节点编辑模式
- 添加子节点：为当前节点添加设备或软件子节点
- 通信节点配置：跳转到流程图页面配置通信协议
- 节点删除：删除当前节点及其所有子节点

#### 2.2.3 详情面板管理
- 详情查看：显示选中节点的详细信息
- 节点编辑：在详情面板中编辑节点属性
- 软件管理：为设备节点添加、编辑、删除软件配置
- 删除操作：删除当前选中的节点

### 2.3 拓扑展示 (topology-display)

#### 2.3.1 网络图可视化
- vis-network引擎：基于vis-network的高性能网络图渲染
- 节点展示：系统、设备、软件节点的图形化展示
- 连接关系：节点间连接关系的可视化
- 实时状态：节点在线/离线状态的颜色区分

#### 2.3.2 交互功能
- 节点选择：点击节点查看详细信息
- 边选择：选择连接关系查看连接详情
- 节点悬停：鼠标悬停显示节点简要信息
- 详情视图：点击节点进入详细信息页面

#### 2.3.3 搜索和筛选
- 全局搜索：按项目名称或设备进行搜索
- 高级筛选：按节点类型（项目/设备/软件）筛选
- 状态筛选：按在线/离线状态筛选节点
- 实时更新：搜索和筛选结果实时更新

### 2.4 报文配置 (packet-config)

#### 2.4.1 报文列表管理
- 报文展示：表格形式展示所有报文配置
- 搜索功能：按报文名称或ID搜索
- 设备筛选：按关联设备筛选报文
- 状态筛选：按启用/禁用状态筛选报文

#### 2.4.2 报文操作功能
- 新建报文：创建新的报文配置
- 编辑报文：进入报文详情编辑页面
- 复制报文：基于现有报文创建副本
- 删除报文：删除不需要的报文配置
- 导出报文：将报文配置导出为JSON文件
- 导入报文：从JSON文件批量导入报文

#### 2.4.3 批量操作
- 批量选择：支持多选报文进行批量操作
- 批量启用：将选中的报文批量设为启用状态
- 批量禁用：将选中的报文批量设为禁用状态
- 批量删除：批量删除选中的报文配置
- 批量导出：批量导出选中的报文配置

#### 2.4.4 报文编辑器
- 字段树管理：左侧显示可用字段类型树
- 拖拽配置：从字段树拖拽字段到协议区域
- 字段排序：在协议区域内拖拽调整字段顺序
- 嵌套字段：支持Struct、Array类型的嵌套字段
- 字段属性编辑：右侧面板编辑字段的详细属性

### 2.5 用户管理 (user)

#### 2.5.1 用户账户管理
- 用户列表：表格形式展示所有用户账户
- 搜索功能：按用户名或邮箱搜索用户
- 角色筛选：按管理员/用户/访客角色筛选
- 状态筛选：按正常/禁用状态筛选用户

#### 2.5.2 用户操作
- 新建用户：创建新的用户账户
- 编辑用户：修改用户信息和权限
- 复制用户：基于现有用户创建新账户
- 删除用户：删除用户账户（需要确认）
- 查看详情：查看用户的完整信息

#### 2.5.3 批量用户操作
- 批量选择：支持多选用户进行批量操作
- 批量启用：批量启用选中的用户账户
- 批量禁用：批量禁用选中的用户账户
- 批量删除：批量删除选中的用户账户

#### 2.5.4 权限管理
- 角色分配：为用户分配不同的角色权限
- 权限配置：详细的功能权限控制
- 状态管理：用户账户的启用/禁用状态控制

### 2.6 系统设置 (settings)

#### 2.6.1 基础配置
- 系统名称：自定义系统显示名称
- 系统描述：系统功能描述信息
- 系统Logo：上传自定义系统Logo

#### 2.6.2 安全设置
- 会话超时：设置用户会话超时时间
- 密码策略：配置密码强度要求
- 登录限制：设置登录失败锁定策略

#### 2.6.3 通知配置
- 邮件通知：启用/禁用邮件通知功能
- 系统通知：配置系统消息通知
- 通知模板：自定义通知消息模板

#### 2.6.4 设置操作
- 保存配置：保存所有系统设置
- 重置设置：恢复系统默认配置

### 2.7 体系层级配置 (hierarchy-settings)

#### 2.7.1 层级管理
- 层级创建：创建新的系统层级类型
- 层级编辑：修改层级的名称、图标、描述等属性
- 层级删除：删除不需要的层级配置
- 层级排序：调整层级的显示顺序

#### 2.7.2 字段配置
- 字段添加：为层级添加自定义字段
- 字段类型：支持文本、数字、日期、选择等字段类型
- 必填设置：设置字段是否为必填项
- 字段删除：删除不需要的自定义字段

#### 2.7.3 层级属性
- 程序名称：层级的程序化标识
- 显示名称：用户友好的显示名称
- 图标选择：选择层级显示图标
- 父级关系：配置层级的父子关系

## 总结

CSSC Node-View是一个功能完整、架构先进的工业级可视化流程设计平台。系统通过模块化的设计，提供了从流程设计、系统建模、拓扑展示到报文配置的全套功能，特别适合于工业物联网、智能制造等领域的复杂系统设计和测试需求。

系统采用现代化的前端技术栈，具有良好的用户体验和扩展性，同时考虑了工业应用对安全性、稳定性和性能的严格要求。通过标准化的组件设计和统一的人机交互模式，为用户提供了直观、高效的系统操作体验。

主要技术亮点包括基于LogicFlow的可视化流程编辑、三层体系结构建模、实时的拓扑展示、灵活的报文配置管理，以及完善的用户权限控制系统。这些功能共同构成了一个强大而易用的工业级系统设计平台。
